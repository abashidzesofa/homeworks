1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

git checkout "filename"

2.Как вернуть файл из индекса в рабочий каталог?

git restore --staged "filename"

3.Как исправить, отменить  ошибочный commit ?

git commit --amend -m "new commit"

4.Как исправить коммит отправленный на сервер GitHub?

git revert HEAD

5.Как создать новую ветку и переключится в нее?

Для создания новой ветки используем команду git brunch "название ветки", переключаемся на нее с помощью команды git checkout "название созданной ветки", лиюо можно сделать все вместе с помощью команды git checkout -b "название ветки"

5.В чем отличение между fast-forward merge  and non fast-forfard merge?

fast-forward merge не создает дополнительного коммита при объединении, non fast-forward merge создает дополнительный коммит в репозитории.

6.Когда может возникнуть конфликт слияния веток, как  его разрешить?

Конфликт модет возникнуть когда несколько веток содержат изменения в одних и тех же строках кода или файлах. Проблема возникает потому, что git не может определить какое из изменений следует выбрать.
Чтобы разрешенить конфликт, нужно в первую очередь разобраться из-за чего и где появился конфликт слияния, нужно разобраться какие изменения стоит внести в файл, с помощью редактора(например). После добавить изменения в индекс с помощью git add и завершить слияние с git merge

7.Можно ли удалить ветку? Как? 

git branch -d "branch name"
